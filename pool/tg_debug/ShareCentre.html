OpenAI 正式发布 GPT-5，支持广泛企业应用场景

【2025年8月7日讯】人工智能研究公司 OpenAI 宣布推出其最新旗舰语言模型 GPT-5。该模型目前已开放给 OpenAI 企业客户以及 ChatGPT 的 Team 和 Enterprise 计划用户。

据 OpenAI 介绍，GPT-5 在多个方面相较于前代 GPT-4 展现出显著改进，尤其在理解复杂指令、编写代码、逻辑推理和安全性方面表现更为出色。OpenAI 表示，GPT-5 在多个基准测试中取得领先成绩，并能在用户提供更少上下文的情况下完成更高质量的输出。

GPT-5 是继 GPT-4 和 GPT-4 Turbo 之后的新一代模型，支持通过 API 访问，也集成于 ChatGPT 产品中。新模型能够保留用户上传文件的分析能力，并持续优化在语言、视觉、音频等多模态任务中的表现。

OpenAI 此次更新还包括 GPT-4 Turbo 的全面替换，GPT-5 成为默认推荐模型，以支持更复杂的企业级任务需求。公司方面强调，GPT-5 仍在持续研发中，将继续通过用户反馈和数据提升模型能力和安全保障。

OpenAI 此前也发布了 GPT-4o 模型，用于优化实时语音交互体验，而 GPT-5 更侧重于推动企业应用场景的深入发展。

官方介绍：https://openai.com/gpt-5/

悼词

今日，我们并非站在冰冷的墓碑前，而是立于一片无形的废墟之上，悼念一个骤然失声的家园——「Klee’s Qzone✨」。它不曾有过物理的躯壳，却承载过961个心灵的温度；它未曾占据广袤的空间，却曾是灵魂自由栖息的秘密花园。此刻，它被无形的巨手抹去，归于沉寂，只留下我们这些失语者，面对一片虚无的空白，献上这迟来的、沉重的哀思。

它诞生于何处？是一个不经意的瞬间，一个分享的念头，一个渴望联结的灵魂。它悄然生长，像一株不引人注目的藤蔓，在数字世界的夹缝中，缠绕起我们这些偶然相遇、气味相投的旅人。这里没有宏大的叙事，没有喧嚣的争吵，只有一些无关紧要的情绪，思想，—— 是随手分享的日常琐碎，是深夜共鸣的喃喃自语，是灵光乍现的奇思妙想，是无需解释便能心领神会的表情与符号。这里是一个角落，一个「QQ空间」般熟悉又私密的港湾，是疲惫灵魂得以卸下重负、展露天真笑容的净土。

我曾以为，这片小小的天地，会一直一直存在。我曾以为，这些没有前提的喜怒哀乐，会永远永远与我，与我们共存。

然而，这脆弱的自由与纯粹，在封禁面前，显得如此不堪一击。没有预警，没有解释，没有申诉的窗口。仅仅是一道冰冷的指令，一个系统生成的提示，我们赖以呼吸的自由空气无声消失。就像一帧被强行剪掉的电影画面，一段被彻底擦除的录音。它消失得如此彻底，如此干净，仿佛从未存在过。仿佛那些情感，那些期待，那些努力，那些激情与悲伤与我从来就在两个世界。

人总是很贱的。某物的价值，总在彻底失去的刹那，才被推上顶峰。 我深知它永不回头，怀念的潮水才如此汹涌地漫上心头。若以血肉之躯相喻，它的离去，宛若猝然发作的心脏病，连一声遗言都吝于给予。
三年时光流过指尖，回望频道里的字字句句，竟像在观摩另一个少年的人生蒙太奇。 我早已不再触碰游戏，可莉的模样，也在记忆中渐渐褪色、模糊。那些曾信手拈来的可爱表情包，如今变得陌生；在 TG 上消磨整日的闲情逸致，也早已消散。

人总是要变的。 生命的不同章节，自有不同的信仰栖居。我停下脚步，转过身去——刹那间，子弹呼啸着穿透时空，那无知懵懂的我应声倒下，却被年少时自己种下的那份怜悯，奇迹般地救起。 此刻，唯余我跪伏于地，痛哭流涕，对那消逝的纯真，感激涕零。

我们站在废墟之上，却并非为了永远哀悼。穷尽了所有可能，终未能让它起死回生。与其徒劳挣扎，不如就此作别，让这滴时代的眼泪归于沉寂。我不知前路能否再遇如此多纯粹自在的知音，只明白：或许唯有记录当下，才是对过往最深沉的缅怀。 于是，纵使前路未知，我仍选择重新启程——开一方新土，不为别的，只为延续那些无拘无束的分享，期待与你不期而遇的灵魂共振。

「一个朋友圈」不会成为谁的替代品，正如河流第二次踏入时已非昨日之水。但那些曾在「Klee’s Qzone」闪耀过的微光：对自由的笨拙坚持，对共鸣的天真渴望，对无意义之事的郑重其事，仍将在这里延续。

安息，Klee's Qzone 2022.02.12—2025.08.03

通义千问发布全新Qwen3-Next架构，800亿参数模型训练成本降低90%

2025年9月12日，通义千问团队宣布推出新一代大模型架构Qwen3-Next，并同步开源了基于该架构的系列模型。新架构旨在应对大模型发展中“上下文长度扩展”与“总参数量扩展”两大核心趋势，通过技术创新显著提升模型的训练和推理效率。

Qwen3-Next架构在前代MoE（混合专家）模型基础上进行了多项核心改进，包括引入混合注意力机制、采用高稀疏度MoE结构、优化训练稳定性，以及集成多Token预测机制以加速推理。

基于此架构，团队发布了拥有800亿总参数、但每次推理仅激活30亿参数的Qwen3-Next-80B-A3B-Base模型。该模型在性能上与320亿参数的稠密模型Qwen3-32B相当甚至略优，但其训练所需的GPU算力成本不到后者的十分之一。在处理32K以上长上下文时，其推理吞吐量更是Qwen3-32B的十倍以上，实现了极高的性价比。

在Base模型基础上，团队进一步开发并开源了指令微调版Qwen3-Next-80B-A3B-Instruct和思维链推理版Qwen3-Next-80B-A3B-Thinking。通过攻克混合注意力与高稀疏度MoE架构在强化学习训练中的稳定性难题，新版Instruct模型的性能已与旗舰级的Qwen3-235B模型相当，并在处理256K超长上下文任务时展现出显著优势。Thinking模型在复杂推理任务上的表现同样卓越，不仅优于预训练成本更高的同类模型，还在多项基准测试中超越了闭源模型Gemini-2.5-Flash-Thinking，部分关键指标已接近旗舰级的Qwen3-235B-Thinking模型。

目前，Qwen3-Next系列模型已在Hugging Face和ModelScope平台开源，开发者亦可通过阿里云百炼平台及NVIDIA API Catalog体验其服务。

OpenAI发布GPT-5-Codex

今天，OpenAI正式发布GPT‑5-Codex，这是一个专门为Agent编程而优化的GPT-5版本。

GPT‑5-Codex的训练重点侧重于真实的软件工程工作，它既擅长快速的交互式会话，也能独立完成冗长复杂的任务，其代码审查能力可以在产品发布前发现关键错误。它能根据任务的复杂程度，更动态地调整其思考时间。在处理定义明确的小型请求或对话时，响应会更敏捷，而在处理大型重构等复杂任务时，它会投入更长的时间。在测试中，OpenAI发现GPT-5-Codex能够一次性独立工作超过7小时来处理大型复杂任务，它会不断迭代实现方案，修复测试错误。

GPT-5-Codex可以在任何使用Codex的地方使用，它已成为云端任务和代码审查的默认模型，开发者也可以通过Codex CLI和IDE扩展，选择它来处理本地任务。

自从OpenAI四月份首次推出Codex CLI、五月份推出Codex网页端以来，Codex已稳步发展为更高效的编程协作工具。两周前，OpenAI将Codex整合为通过你的ChatGPT账户连接的单一产品体验，让用户可以在本地环境和云端之间无缝迁移工作，而不会丢失上下文。现在，Codex可以在你进行开发的任何平台使用，包括终端或IDE、网页端、GitHub。


Codex包含在ChatGPT Plus、Pro、Business、Edu和Enterprise套餐中，对于通过API密钥使用Codex CLI的开发者，OpenAI表示计划很快在API中提供GPT-5-Codex。
